# Rapport de couverture e2e des routes backend

Ce rapport recense les routes backend que j'ai pu retrouver dans les tests e2e existants, puis liste les routes pour lesquelles je n'ai trouvé aucune reference dans `transcendence/frontend/interface/tests/e2e`.

## Methode

- J'ai considere une route comme couverte si elle apparait explicitement dans un `page.route(...)` ou dans un helper WebSocket utilise par un test e2e.
- J'ai distingue les routes HTTP des routes WebSocket.
- Les routes generées par `django_prometheus` ne sont pas testables ici de facon fiable sans connaitre les chemins exacts exposes par le package, donc je les laisse hors matrice de couverture.

## Lecture rapide

- La couverture e2e est bonne sur les parcours critiques: login, 2FA, logout, profil, presence, chat, invitations, matchmaking et partie.
- Les zones les plus clairement non couvertes sont le recovery password, l'inscription locale, la gestion des amis via les routes d'action, et les routes HTTP de chat qui ne passent pas par le WebSocket.
- Si tu veux, le prochain pas utile est de transformer ce rapport en checklist de priorisation des tests manquants.
